# 📌 한화 SW캠프 22기 최종 프로젝트 워크스페이스

이곳은 각 팀의 프로젝트 코드를 안전하게 관리하고, 최종 CI/CD 파이프라인 구축 및 메인 저장소(Main Repository)와 연동하기 위한 통합 Organization입니다.

원활한 프로젝트 진행과 관리를 위해 아래의 **레포지토리 생성 규칙 및 서브모듈 연결 가이드**를 반드시 숙지하고 준수해 주세요.

---

## 📐 1. 레포지토리(Repository) 생성 및 네이밍 규칙

새로운 레포지토리를 생성할 때는 **각 팀의 팀장(혹은 인프라 담당자) 1명만 생성**해야 하며, 파편화를 막기 위해 아래의 네이밍 규칙을 엄격히 지켜주세요.

* **네이밍 포맷:** `팀명-도메인`
* **Visibility:** 프로젝트 기간 중에는 가급적 **Private**으로 생성합니다.

**✅ 올바른 생성 예시**
* `team1-frontend` (1팀 프론트엔드)
* `team1-backend` (1팀 백엔드)
* `team2-infra` (2팀 인프라/설정)

**❌ 잘못된 생성 예시**
* `frontend` (어느 팀인지 알 수 없음)
* `team1_FrontEnd` (대소문자 및 언더바 혼용 지양)
* `my-awesome-project` (개인 프로젝트 명칭 지양)

---

## 🤝 2. 팀원 초대 및 권한 부여 방법

레포지토리를 생성한 사람은 해당 레포지토리의 **Admin(관리자)** 권한을 갖게 됩니다. 생성 직후 본인의 팀원들을 초대하여 코드를 함께 작성할 수 있도록 세팅해 주세요.

1. 생성한 레포지토리의 **[Settings]** 탭으로 이동
2. 좌측 메뉴에서 **[Collaborators and teams]** 클릭
3. **[Add people]** 버튼을 눌러 팀원들의 GitHub 아이디 검색 및 초대
4. 권한(Role)은 코드를 푸시할 수 있도록 **`Write`** 로 설정
> 💡 **주의:** 기본적으로 다른 팀의 레포지토리는 코드를 열람(Read)할 수는 있지만, 수정(Push)은 불가능하도록 안전하게 격리되어 있습니다. 본인 팀의 레포지토리에만 작업해 주세요!

---

## 🔗 3. 메인 저장소에 서브모듈(Submodule) 연결하기

각 팀의 레포지토리 세팅이 끝났다면, 강사님이 제공한 **메인 프로젝트 레포지토리**에 우리 팀의 레포지토리를 서브모듈로 연결해야 합니다.

### Step 1. 메인 레포지토리 Clone (로컬)
먼저 메인 저장소를 본인의 컴퓨터로 클론하고 해당 폴더로 이동합니다.
```bash
git clone [메인_레포지토리_주소]
cd [메인_레포지토리_폴더명]
```

### Step 2. 서브모듈 추가
우리 팀이 사용할 폴더 위치(예: app/frontend)에 방금 만든 Organization 내 레포지토리를 서브모듈로 추가합니다.

```bash
git submodule add [생성한_우리팀_레포지토리_URL] [경로/폴더명]

# 예시:
# git submodule add https://github.com/swcamp-22th-projects/team1-frontend.git app/frontend
```

### Step 3. 🚨 [중요] 브랜치 체크아웃
서브모듈은 처음 추가 시 특정 브랜치가 아닌 Detached HEAD 상태로 연결됩니다. 코드 작업을 시작하기 전에 반드시 서브모듈 폴더로 이동하여 main 브랜치로 체크아웃 하세요.

```bash
cd [서브모듈_경로]   # 예: cd app/frontend
git checkout main
```

### Step 4. 메인 레포지토리에 변경사항 반영
다시 메인 레포지토리의 최상위 경로로 돌아와서 서브모듈 연결 내역을 커밋하고 푸시합니다.

```bash
cd ..
git add .
git commit -m "feat: Team X 서브모듈 연결"
git push origin main
```
